# علاقات قاعدة البيانات + تدفق التدقيق (Relations + Flow)

مستند **منفصل** عن [خريطة-التدقيق-والمنطق.md](./خريطة-التدقيق-والمنطق.md): الأول يشرح **المعنى والمنطق والـ Flow التفصيلي**؛ هذا الملف يركّز على **علاقات الجداول (Relations)** و**مخطط تدفق مختصر** يربط الكيانات ببعضها.

---

## 1. نطاق هذا المستند

| يشمل | لا يشمل بالتفصيل |
|------|------------------|
| جداول الدورات المالية، الكاش، التدقيق، المؤجل، الدفتر، لقطة الصندوق | شحن، معتمدين، صناديع (إلا إذا لامست الدورة) |
| ربط الوكالة الفرعية بـ `member_user_id` لحساب W | واجهات المستخدم |

## 2. مخطط علاقات (ER) — نواة التدقيق والدورة

العلاقات **المنطقية** (بدون `FOREIGN KEY` في PostgreSQL) تُشارك بخط منقط في الوصف؛ الـ **FK الصريحة** من `schema.pg.sql` تُذكر في الجدول التالي.

```mermaid
erDiagram
  users {
    int id PK
    string username
  }

  financial_cycles {
    int id PK
    int user_id "مالك الدورة"
    string name
    text management_data
    text agent_data
    real transfer_discount_pct
  }

  payroll_cycle_cache {
    int user_id "جزء من PK"
    int cycle_id "جزء من PK"
    text management_data
    text agent_data
    text audited_agent_ids
    text audited_mgmt_ids
  }

  payroll_cycle_columns {
    int user_id "جزء من PK"
    int cycle_id "جزء من PK"
    string agent_salary_col
    string mgmt_user_id_col
  }

  payroll_user_audit_cache {
    int user_id "جزء من PK"
    int cycle_id "جزء من PK"
    string member_user_id "جزء من PK"
    string audit_status
    string audit_source
  }

  deferred_salary_lines {
    int id PK
    int user_id
    int cycle_id
    string member_user_id
    real balance_d
    real salary_before_discount
  }

  ledger_entries {
    int id PK
    int user_id
    string bucket
    string source_type
    int cycle_id "FK اختياري"
  }

  cash_box_snapshot {
    int id PK
    int cycle_id
    real cash_balance
    real source_y_z
    real source_first_sheet_w
  }

  shipping_sub_agencies {
    int id PK
    real company_percent
  }

  user_agency_link {
    int id PK
    string member_user_id UK
    int sub_agency_id
  }

  agency_cycle_users {
    int id PK
    int cycle_id
    int sub_agency_id
    string member_user_id
    real base_profit_w
  }

  users ||--o{ financial_cycles : "user_id"
  financial_cycles ||--|| payroll_cycle_cache : "PK user_id + cycle_id"
  financial_cycles ||--|| payroll_cycle_columns : "PK user_id + cycle_id"
  financial_cycles ||--o{ payroll_user_audit_cache : "cycle_id"
  financial_cycles ||--o{ deferred_salary_lines : "cycle_id"
  financial_cycles ||--o{ ledger_entries : "cycle_id FK"
  financial_cycles ||--o{ cash_box_snapshot : "cycle_id منطقي"
  users ||--o{ payroll_user_audit_cache : "user_id"
  users ||--o{ deferred_salary_lines : "user_id"
  users ||--o{ ledger_entries : "user_id"
  shipping_sub_agencies ||--o{ user_agency_link : "sub_agency_id"
  financial_cycles ||--o{ agency_cycle_users : "cycle_id منطقي"
  shipping_sub_agencies ||--o{ agency_cycle_users : "sub_agency_id FK"
```

### 2.1 مفاتيح وقيود (ملخص)

| الجدول | المفتاح | ملاحظة |
|--------|---------|--------|
| `financial_cycles` | `id` | مركز الدورة؛ `user_id` → `users.id` (منطقي في التطبيق). |
| `payroll_cycle_cache` | `(user_id, cycle_id)` | لقطة صفوف للبحث/التدقيق؛ `cycle_id` يشير لـ `financial_cycles.id`. |
| `payroll_cycle_columns` | `(user_id, cycle_id)` | أعمدة A/D/… لكل دورة. |
| `payroll_user_audit_cache` | `(user_id, cycle_id, member_user_id)` | حالة التدقيق لكل عضو في الدورة. |
| `deferred_salary_lines` | `UNIQUE(user_id, cycle_id, member_user_id)` | مؤجل بعد خصم التحويل؛ يُستبدل عند إعادة بناء **تلك الدورة** فقط. |
| `ledger_entries` | `id` | `cycle_id` → `financial_cycles(id) ON DELETE SET NULL` (في المخطط). |
| `cash_box_snapshot` | `id` | مرتبط بـ `cycle_id` **بدون FK** في الملف؛ الاستخدام منطقي. |
| `user_agency_link` | `member_user_id` UNIQUE | لربط مستخدم إداري بوكالة فرعية (يؤثر على حساب W). |
| `agency_cycle_users` | فريد `(cycle_id, sub_agency_id, member_user_id)` | قاعدة مزامنة W للدورة. |

## 4. Flow — ربط Relations بالعمليات (مختصر)

يُظهر كيف تتحرك البيانات بين الكيانات نفسها دون تكرار شرح [خريطة-التدقيق-والمنطق.md](./خريطة-التدقيق-والمنطق.md).

```mermaid
flowchart LR
  subgraph cycle["الدورة financial_cycles"]
    FC[(id + بيانات الجداول)]
  end

  subgraph cache["لقطات البحث"]
    PCC[payroll_cycle_cache]
    PCCOL[payroll_cycle_columns]
  end

  subgraph audit["تدقيق الأعضاء"]
    PUA[payroll_user_audit_cache]
  end

  subgraph money["أثر مالي"]
    DSL[deferred_salary_lines]
    LED[ledger_entries]
    CBS[cash_box_snapshot]
  end

  subgraph link["وكالات فرعية"]
    UAL[user_agency_link]
    SSA[shipping_sub_agencies]
  end

  FC --> PCC
  FC --> PCCOL
  FC --> PUA
  FC --> DSL
  FC --> LED
  FC --> CBS
  UAL --> SSA
  FC -.->|حساب W منقسم| UAL
```

**قراءة سريعة:**

1. **`financial_cycles`** يغذّي الكاش والأعمدة والتدقيق والمؤجل.
2. **`payroll_user_audit_cache`** يحدّد مَن **مدقق**؛ مع **`deferred_salary_lines`** يُستبعد المدقق من المؤجل عند إعادة البناء.
3. **`ledger_entries`** يستقبل `audit_management_yz` / `audit_management_w` و`transfer_discount_profit` (حسب المسار).
4. **`user_agency_link` + `shipping_sub_agencies`** يدخلان في **حساب عمود W** (حصة الشركة) عند تجميع الإدارة وليس كجدول تدقيق مباشر.

## 5. Sequence مختصر — من `cycle_id` إلى الجداول

```mermaid
sequenceDiagram
  participant C as cycle_id
  participant FC as financial_cycles
  participant PCC as payroll_cycle_cache
  participant PUA as payroll_user_audit_cache
  participant DSL as deferred_salary_lines
  participant L as ledger_entries

  C->>FC: SELECT / UPDATE بيانات الدورة
  C->>PCC: لقطة صفوف للبحث
  C->>PUA: حالة مدقق لكل member_user_id
  C->>DSL: أسطر مؤجل (غير مدقق)
  C->>L: قيود net_profit مرتبطة بالدورة
```
